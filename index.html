JT's Website Design System

Colors

The colors shown in this system were selected from a split-complements color scheme.

Name Value Usage
Off-White color: hsl(135, 0%, 95%) Used to format the background of the web page (outside of dark mode styles). The background of this web page is set to this color.
Spring Green color: hsl(135, 90%, 80%) As one of the two primary colors used to stylize the web page.
Future Blue color: hsl(200, 90%, 80%) As one of the two primary colors used to stylize the web page. Used in place of Spring Green when two nearby elements need to stand out from each other.
Vibrant Red color: hsl(350, 90%, 80%) Provides contrast to the other colors, used to grab the user's attention and highlight important elements on the page.

Font Dimensions

The guidelines below were selected based on a golden ratio calculation. Fonts on this page are styled using these guidelines. All font dimensions are selected to work with a line height of 26px.

Tag Name Font Family Options Font Size Font Weight Usage
h1 Sans-Serif (Modern), Serif (Traditional and often easier to read), or Monospace or Cursive (Stylized) 26px 700 Used for article titles and in the website header
h2 - h6 Same as that used for the paragraph. Alternatively, pair a Serif paragraph with Sans-Serif headers (as done on this page) or vice versa. 20px 700 Used for headers in the website body.
p Sans-Serif (Modern) or Serif (Traditional and often easier to read). 16px 400 Used for body text in the paragraph.
blockquote, aside Same as that used for the paragraph, or choose: Serif (Traditional and often easier to read), or Monospace or Cursive (Stylized). 20px 400 Used for blockquotes (long quotations placed within an indented section within an article) or asides (either text from the article that is repeated due to its importance, or "footnote"-type text that provides contextual information outside the flow of the writing).
pre Select a popular source code font (e.g. Source Code Pro), and/or use the Brush Script MT Web-Safe CSS Font Stack. 14px 400 Used for preformatted text, like when demonstrating code in a code block.

Notes on Text Formatting

  • Always prefer ragged-right text margins over justified margins, as they are easier to read.
  • Inline-block formatting can be used to limit the effects of background-color on text to the space taken up by the text itself.
  • All content should be centered, but allowed to take up 75 to 90% of the width of the screen.
  • Select the ul or ol tags as opposed to their child tags when applying alignment rules.
  • Add top-margin to headers to keep text spaced appropriately, accounting for margin collapse.
